================================================================================
PHASE 3: SPEECH-TO-TEXT (STT) VIA WHISPER API
================================================================================

Step 3.1: Create Whisper Edge Function
---------------------------------------
File: supabase/functions/speech-to-text/index.ts

Purpose:
  - Receive audio file from client (base64 or multipart)
  - Call OpenAI Whisper API for transcription
  - Return transcribed text

API Details:
  - Endpoint: POST /functions/v1/speech-to-text
  - Input: { audio: base64String, format: 'mp4' | 'webm' | 'm4a' }
  - Output: { success: boolean, text?: string, error?: string }
  - Whisper model: whisper-1
  - Language: auto-detect

Cost:
  - $0.006 per minute of audio
  - Average message: 3-5 seconds = ~$0.0005 per transcription

Step 3.2: Create Voice Recording Hook
--------------------------------------
File: hooks/useVoiceRecording.ts

Purpose:
  - Manage recording state and permissions
  - Record audio using expo-av
  - Upload audio to Whisper Edge Function
  - Return transcribed text

Key Functions:
  - requestPermissions(): Request microphone permission
  - startRecording(): Begin audio capture
  - stopRecording(): End capture and transcribe
  - State: isRecording, duration, transcribedText, error

Implementation:
  - Use Audio.Recording from expo-av
  - Record in .m4a format (iOS) or .webm (Android)
  - Set audio mode for recording
  - Convert recording to base64
  - Call speech-to-text Edge Function
  - Handle errors gracefully

Step 3.3: Create Voice Recorder Component
------------------------------------------
File: components/chat/voice-recorder.tsx

Purpose:
  - Visual UI for recording audio
  - Show recording animation/waveform
  - Display recording duration
  - Cancel or send recording

UI Elements:
  - Pulsing red recording indicator
  - Duration timer (00:05)
  - Cancel button (X)
  - Send button (checkmark)
  - Waveform or volume visualization (optional)

Step 3.4: Update ChatInputBar Component
----------------------------------------
File: components/chat/chat-input-bar.tsx

Changes:
  1. Add microphone button next to send button
  2. Show VoiceRecorder component when recording
  3. Hide text input when recording
  4. Auto-populate text input with transcribed text
  5. Disable microphone button during AI response

UI States:
  - Normal: Text input + mic button + send button
  - Recording: VoiceRecorder component (full width)
  - Transcribing: Loading indicator

================================================================================
PHASE 5: CONFIGURATION & DEPLOYMENT
================================================================================

Step 5.1: Configure App Permissions
------------------------------------
File: app.json

Add to expo config:
  "plugins": [
    [
      "expo-av",
      {
        "microphonePermission": "Allow Orchestra to access your microphone for voice input."
      }
    ]
  ]

iOS: NSMicrophoneUsageDescription
Android: RECORD_AUDIO permission

Step 5.2: Deploy Whisper Edge Function
---------------------------------------
Commands:
  npx supabase functions deploy speech-to-text

Note: OpenAI API key already set as OPENAI_API_KEY secret

Step 5.3: Test on Physical Device
----------------------------------
Why: Microphone doesn't work on iOS Simulator or some Android emulators

Test Cases:

  3. Record long message (30 seconds)
  4. Cancel recording mid-way
  5. Background noise handling
  6. Multiple recordings in sequence
  7. TTS playback of short message
  8. TTS playback of long message
  9. Stop TTS mid-speech
  10. Multiple TTS playbacks

PHASE 6: POLISH & OPTIMIZATION
================================================================================

Step 6.1: Add Voice Settings
-----------------------------
Optional: Add to support screen

  - TTS pitch

Step 6.2: Error Handling
-------------------------
Handle these scenarios:
  - Microphone permission denied
  - Whisper API failure (429, 500)
  - Audio recording failure
  - Network timeout during transcription
  - TTS failure on device
  - Unsupported audio format

User-friendly messages:
  - "Microphone access required for voice input"
  - "Transcription failed. Please try again."
  - "Voice playback not available on this device"

Step 6.3: Performance Optimization
-----------------------------------
  - Compress audio before upload (reduce file size)
  - Cache common phrases for faster response
  - Use streaming if OpenAI supports it
  - Debounce recording start (prevent accidental taps)

================================================================================
TECHNICAL SPECIFICATIONS
================================================================================

Audio Recording Settings:
  - Format: AAC (m4a) on iOS, WebM on Android
  - Quality: Medium (balance size and quality)
  - Sample rate: 16kHz (Whisper recommendation)
  - Channels: Mono
  - Bit rate: 64kbps

TTS Settings:
  - Rate: 1.0 (normal speed, adjustable)
  - Pitch: 1.0 (normal pitch)
  - Language: Auto-detect from device
  - Volume: 1.0 (max)

Whisper API:
  - Model: whisper-1
  - Language: Auto-detect (or specify 'en')
  - Response format: json
  - Temperature: 0 (deterministic)

================================================================================
COST ANALYSIS
================================================================================

Speech-to-Text (Whisper):
  - Cost: $0.006 per minute
  - Average message: 4 seconds = $0.0004
  - 1,000 voice messages = $0.40
  - 10,000 voice messages = $4.00

Text-to-Speech (expo-speech):
  - Cost: FREE (uses device's built-in TTS)
  - No API calls needed
  - No token usage

Combined (with existing chat):
  - Voice input: $0.0004/message
  - AI response: $0.0001/message (GPT-4o-mini)
  - Voice output: $0.00/message
  - Total per voice interaction: ~$0.0005

Expected monthly cost (1,000 users, 10 messages/user/month):
  - 10,000 messages × $0.0005 = $5.00/month
